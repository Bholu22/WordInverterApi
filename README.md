# Word Inverter API

A **.NET 9 REST API** that inverts each word in a sentence and stores every request/response pair in **SQL Server** using **Entity Framework Core**. API documentation is served via **Scalar UI**.

> Example: `"abc def ghi"` → `"cba fed ihg"`

---

## Prerequisites

Before running this project, make sure you have the following installed on your machine:

| Tool | Version | Download |
|------|---------|----------|
| .NET SDK | 9.0 or later | https://dotnet.microsoft.com/download/dotnet/9.0 |
| SQL Server | Any edition (Developer/Express are free) | https://www.microsoft.com/en-us/sql-server/sql-server-downloads |
| EF Core CLI tools | Latest | Run command below |

**Install EF Core CLI tools** (run once globally):
```bash
dotnet tool install --global dotnet-ef
```

**Verify your setup:**
```bash
dotnet --version       # Should show 9.x.x
dotnet ef --version    # Should show a version number
```

## Step 1 — Clone or copy the project

If you received a zip file, extract it. If it's in a Git repository:

```bash
git clone <repository-url>
cd WordInverterApi
```

---

## Step 2 — Configure the database connection

Open `appsettings.json` and update the connection string to point to your SQL Server instance.

### Option A — Windows Authentication (most common on Windows)
```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost;Database=WordInverterDb;Trusted_Connection=True;TrustServerCertificate=True;"
  }
}
```

### Option B — SQL Server username & password
```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost;Database=WordInverterDb;User Id=sa;Password=YourPasswordHere;TrustServerCertificate=True;"
  }
}
```

> **Note:** You do **not** need to manually create the `WordInverterDb` database.
> It will be created automatically in the next step.

---

## Step 3 — Restore NuGet packages

```bash
dotnet restore
```

---

## Step 4 — Run database migrations

This creates the database and the `InversionRecords` table automatically:

```bash
dotnet ef database update
```

You should see output ending in:
```
Applying migration '..._InitialCreate'.
Done.
```

> If you get a connection error here, double-check your connection string in `appsettings.json`
> and make sure SQL Server is running on your machine.

## Step 5 — Run the API

```bash
dotnet run
```

You will see output like:

## API Endpoints

### POST `/api/wordinverter/invert`
Inverts each word in the provided sentence and saves the pair to the database.

**Request body:**
```json
{
  "sentence": "abc def ghi"
}
```

**Response:**
```json
{
  "id": 1,
  "originalSentence": "abc def ghi",
  "invertedSentence": "cba fed ihg",
  "createdAt": "2024-01-15T10:30:00Z"
}
```

---

### GET `/api/wordinverter/all`
Returns the full list of all stored request/response pairs, newest first.

**Response:**
```json
[
  {
    "id": 2,
    "originalSentence": "hello world",
    "invertedSentence": "olleh dlrow",
    "createdAt": "2024-01-15T10:35:00Z"
  },
  {
    "id": 1,
    "originalSentence": "abc def ghi",
    "invertedSentence": "cba fed ihg",
    "createdAt": "2024-01-15T10:30:00Z"
  }
]
```

---

### GET `/api/wordinverter/find?word={word}`
Finds all pairs where the given word appears in either the original or inverted sentence.

**Example:**
```
GET /api/wordinverter/find?word=abc
```

**Response:** Same shape as `/all` but filtered to matching records only.

## Troubleshooting

### `dotnet ef` command not found
```bash
dotnet tool install --global dotnet-ef
# Then close and reopen your terminal
```

### Cannot connect to SQL Server
- Make sure SQL Server service is running (check Windows Services or SQL Server Configuration Manager)
- Verify the `Server=` value in your connection string matches your SQL Server instance name
- For named instances use: `Server=localhost\SQLEXPRESS` instead of `Server=localhost`

### Port already in use
Edit `Properties/launchSettings.json` and change the port numbers under `applicationUrl`.

### Migrations folder is missing
If the `Migrations/` folder was not included, regenerate it:
```bash
dotnet ef migrations add InitialCreate
dotnet ef database update
```

### SSL certificate error in browser
Run this once to trust the .NET dev certificate:
```bash
dotnet dev-certs https --trust
```
